# 📅 2026.02.25 - [Java] 람다식과 함수형 인터페이스 이해

## 📌 학습 주제
- 람다식(Lambda Expression)
- 함수형 인터페이스
- 람다식을 매개변수로 전달하기

---

## 🔎 핵심 개념

- **함수형 인터페이스**란 추상 메서드가 단 하나만 존재하는 인터페이스이다.
- 예를 들어, 단 하나의 `calculate()` 추상 메서드를 가진 `Calculable` 인터페이스는 함수형 인터페이스이다.
- 람다식은 해당 추상 메서드를 구현하는 익명 구현 객체를 간결하게 표현한 것이다.
- 매개변수 타입이 함수형 인터페이스라면, 메서드 호출 시 람다식을 전달할 수 있다.
- 즉, **행동(로직)을 매개값으로 전달하는 방식**이다.

---

## 💻 예제 코드

### ✅ 함수형 인터페이스 정의

```java
@FunctionalInterface
interface Calculable {
    void calculate(int x, int y);
}
```

---

### ✅ 람다식 사용 예제

```java
public class LambdaExample {

    public static void action(Calculable calculable) {
        calculable.calculate(10, 20);
    }

    public static void main(String[] args) {

        // 람다식 전달
        action((x, y) -> {
            int result = x + y;
            System.out.println("결과: " + result);
        });
    }
}
```

---

## ⚠️ 헷갈렸던 부분 / 실수

- 람다식은 인터페이스 자체를 구현하는 것이 아니라,  
  **추상 메서드를 구현하는 것**이라는 점이 처음에는 헷갈렸다.
- 함수형 인터페이스는 반드시 추상 메서드가 하나여야 하며,  
  그렇지 않으면 람다식을 사용할 수 없다.
- 람다식은 객체가 아니라 “익명 구현 객체의 축약 표현”이라는 점을 이해하는 데 시간이 걸렸다.

---

## 🔄 Before → After 비교

### 🔹 기존 익명 구현 객체 방식

```java
action(new Calculable() {
    @Override
    public void calculate(int x, int y) {
        System.out.println(x + y);
    }
});
```

---

### 🔹 람다식 적용 후

```java
action((x, y) -> System.out.println(x + y));
```

👉 불필요한 클래스 선언이 사라지고 코드가 훨씬 간결해졌다.

---

## 🧠 정리

- 람다식은 함수형 인터페이스와 함께 사용된다.
- 함수형 인터페이스는 **추상 메서드가 하나만 존재**해야 한다.
- 람다식은 해당 추상 메서드를 구현하는 코드를 간단하게 표현한 것이다.
- 매개변수 타입이 함수형 인터페이스라면, 메서드 호출 시 람다식을 전달할 수 있다.
